Welcome to the AMA Style Guide

Click the "all" link or navigation above to browse styles. You can search for patterns using "Find Pattern" in the gear on the right.

Working with our style guide

Structure

This living style guide is built with Pattern Lab and adheres to the principles of Atomic Design.

Issues

If you have questions or encounter an issue working with the style guide, report it as a Github issue.

Contributions

For any code that you would like to contribute to the style guide, first create an issue in Github explaining clearly what problem you are trying to solve.

Make sure you have adhered to the guidelines outlined in the Code Contributions documentation that lives in the /docs folder of this project before submitting a pull request. Reference the Github issue you created when you submit your pull request.

Annotation and Documentation

Each pattern should have its own accompanying Markdown file that contains 1) a useful description of the pattern and 2) links to the relevant Jira tickets/Github issues relating to that pattern's creation. This information is needed to provide context as to how and when a particular pattern should be used.
